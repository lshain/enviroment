    CONFIG_SUSPEND is declared in autoconf.h, this file must be seen by semantic to be able to recongnize all the defines there. No need to define that macro manually !

    This can be done in emacs configuration file by adding this line:

    (add-to-list 'semantic-lex-c-preprocessor-symbol-file "/home/abdellatif/android/kernel/include/generated/autoconf.h")

  • Alex&nbsp;Ott

    Thanks for comment. I'll try to clarify some issues in future versions of this article. About questions:
    - cpp-root-project is needed, if you want to supply additional information for semantic, such as project-specific include paths, preprocessor variables, etc. Examples you can see in middle of my config - http://github.com/alexott/emac... I also use it to store some additional information, such as command for compilation, etc.
    - for boost, you need to add boost/config.hpp into list of files with preprocessor directives, for example, (add-to-list 'semantic-lex-c-preprocessor-symbol-file "/usr/include/boost/config.hpp"), this should properly expand all boost macroses. And it's better to use latest version of CEDET from CVS - there are many bugs were fixed in it
    - about 'No Global support' - it's just a warning, CEDET uses gtags database to provide more faster jumps, but it also works without it, so you can ignore these messages, or build a database in /usr/include
    - for idle service error message - please, look into CEDET manual for description how to debug such errors, and after getting more information, send it into CEDET mailing list

                                                      Thanks for comment. I'll try to clarify some issues in future versions of this article. About questions:
                                                      - cpp-root-project is needed, if you want to supply additional information for semantic, such as project-specific include paths, preprocessor variables, etc. Examples you can see in middle of my config - http://github.com/alexott/emac... I also use it to store some additional information, such as command for compilation, etc.
                                                      - for boost, you need to add boost/config.hpp into list of files with preprocessor directives, for example, (add-to-list 'semantic-lex-c-preprocessor-symbol-file "/usr/include/boost/config.hpp"), this should properly expand all boost macroses. And it's better to use latest version of CEDET from CVS - there are many bugs were fixed in it
                                                      - about 'No Global support' - it's just a warning, CEDET uses gtags database to provide more faster jumps, but it also works without it, so you can ignore these messages, or build a database in /usr/include
                                                      - for idle service error message - please, look into CEDET manual for description how to debug such errors, and after getting more information, send it into CEDET mailing list
